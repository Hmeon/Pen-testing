Metasploit 기반 취약점 재현 및 탐지 포인트 도출

가상화 실습 환경에서 취약점 경로를 재현하고 Packet/Log Evidence로 탐지 포인트를 도출했습니다. 채용관이 빠르게 판단할 수 있도록 스냅샷 지표와 원문 미리보기를 분리해, 근거 확인 흐름이 즉시 읽히도록 구성했습니다.

  • 실습 환경: VirtualBox 기반 이중 VM(공격자/대상)
  • 분석 대상: 레거시 취약 서비스 + 사용자 실행형 경로

F-01 레거시 FTP 서비스 노출

서버 측 경로에서 세션 성립이 확인된 고위험 항목으로, 취약 서비스 제거와 접근 원천 제한이 우선 과제입니다.

CWE-912CVSS v3.1 9.8HighOpen

근거: IMG-001, IMG-002

F-02 사용자 실행형 파일 통제 미흡

사용자 실행 전제에서 역방향 세션이 성립되어, 파일 실행 통제 및 최소 권한 운영 강화가 필요합니다.

CWE-494CVSS v3.1 8.1HighOpen

근거: IMG-016

F-03 이그레스 통제 부재

비인가 콜백 트래픽 가능성이 관측되어, 아웃바운드 필터링 및 예외 승인 절차 분리가 필요한 상태입니다.

CWE-284CVSS v3.1 6.5MediumPlanned

근거: IMG-003

Method

수행 프레임워크

재현 가능한 절차와 근거 추적을 위해 아래 순서로 테스트를 설계하고 수행했습니다.

  1. 준비 및 범위 정의

    허가 범위, 실습 조건, 테스트 목표를 사전에 고정해 범위 이탈을 방지합니다.

  2. 취약 경로 재현

    레거시 서비스 경로와 사용자 실행형 경로를 분리해 각각의 성립 조건을 검증합니다.

  3. Packet/Log 증거 확보

    패킷 캡처와 실행 로그를 연결해 탐지 관점에서 재해석 가능한 Evidence를 구성합니다.

  4. 리스크 우선순위화

    Finding별 CWE/CVSS 기준으로 개선 우선순위를 정하고 후속 재검증 계획을 수립합니다.
