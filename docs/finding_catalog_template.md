# Finding Catalog Template (CVSS v4 + CWE + OWASP Top10:2025)

목적: 보고서 내용을 재사용 가능한 보안 결과물 형식으로 표준화한다.

## 1) 필수 필드 정의

| 필드 | 설명 | 예시 |
|---|---|---|
| Finding ID | 고유 식별자 | `PT-2026-001` |
| 제목 | 취약점 핵심 제목 | 인증 우회 가능성 |
| 요약 | 2~3문장 요약 | 특정 입력 검증 부재로 우회 발생 |
| 심각도 | 조직 기준 등급 | Critical / High / Medium / Low |
| CVSS v4 Vector | 표준 벡터 | `CVSS:4.0/...` |
| CVSS v4 Score | 수치 점수 | `8.7` |
| CWE | 취약점 분류 | `CWE-287` |
| OWASP Top10:2025 | 카테고리 | `A07:2025` |
| 영향(C/I/A) | 기밀/무결성/가용성 영향 | C:High, I:High, A:Low |
| 재현 조건 | 사전 조건 | 계정 권한, 네트워크 범위 |
| 재현 절차 | 단계별 수행 | Step 1~N |
| 증거 | 캡처/로그/패킷 | `img/fig_6-3.png` |
| 원인 | 기술적 원인 | 서버 측 검증 누락 |
| 개선안 | 수정 권고 | 검증 로직 추가/권한 재설계 |
| 검증 결과 | 조치 후 상태 | 패치 후 재현 실패 |
| ATT&CK (옵션) | TTP 매핑 | `T1190` |
| KEV (옵션) | 실제 악용 여부 | 해당 없음 / 해당 |

## 3) 우선순위 규칙 (권장)

1. `CVSS v4 >= 9.0`은 무조건 P0
2. `KEV 해당`은 점수와 무관하게 P0 상향 검토
3. 인증/권한 우회 계열은 서비스 영향도 기준으로 추가 상향

## 4) 본 프로젝트 적용 메모

1. 최소 3건을 우선 작성한다.
2. 모든 Finding은 최소 1개 이상의 `img/fig_*` 증거와 연결한다.
3. PDF 본문 표현과 HTML 요약 표현의 용어를 통일한다.
